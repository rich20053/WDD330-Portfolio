  • Notes:

    Master Mobil UX

    • Mobile First - easier to build small and make bigger than the other way around
    • Mobile first sometimes requires a mindset change
    • Doing otherwise is like trying to put a marshmallow in a piggy bank
    • Mobile devices are proliferating quickly - this is not a fad
    • Three things you can do to make a difference:
      1. Be There - to help move someone along their decision journey
      2. Be Useful - satify their needs - provide them with what they came for as quickly as possible
      3. Be Quick - They should not notice how quick it is
    • Design so that actions are easily reachable with the user's thumb
    • Networks matter
    • Speed matters
      • Reduce image usage throughout sites/apps
      • SVG/Webfonts are lower bandwidth
      • Optimize and minify CSS and JS files
      • GZIP files to eliminate unnecessary data
    • Software is mediocre - we make do
    • Placement matters - menu at bottom?
    • Reachability produces perception of speed improvement
    • Strive for "amazing" rather than settling for "good enough"
    • "Menu" in a button produced better user understanding
    • Mobile is a great forcing function for simplicity
    • Mobile Forms
      • Save space by putting labels inside input
      • Use float labels to keep a small version of the label above the entry
      • Make boxes larger to make them easier to hit with the thumb, including checkboxes, etc.
      • Only include what is actually needed so that the fields do not go beyond the screen
      • Only display fields as user identifies the information needed, like contact method
      • Label should identify what information is needed, less generic
      • Enable "Autocomplete"
    • Passwords Matter
      • Have a forgot password link
      • Show passwords, but have a button to hide, if needed. Hiding the password hides typos and can cause frustration.
    • Credit Card information
      • Use auto complete
      • Use good spacing and auto advanced
    • Always provide the best keyboard for the information they are entering
    • Use a big key keypad, when possible
    • Link phone numbers so that a click will initiate calls
    • Advertising is competing for space
    • Make ads small and simple
    • Allow expansion on click
    • Small add at bottom with a window to the ad behind the content
    • Stop using pop ups and pop overs
    • Remember that users do not visit your website for the advertisements

  • Questions:

    1. Is localStorage a popular way to persist data in real world applications? Or is it just a work around when we don't want to deal with sessions, or even an actual database? I think this question in the example is a good one. I think you want to use local storage when the data is not important to you and your website, but may be important to the user.
